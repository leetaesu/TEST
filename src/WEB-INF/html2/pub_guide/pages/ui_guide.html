Publishing guide

개요

  • 본 가이드는 키움 마이데이터 모바일의 퍼블리싱에 대해서 일관적으로 작성하는 방법을 제공하고자 합니다.
  • 퍼블리싱 개발은 프론트-엔드 페이지의 기본 골격을 형성하기 때문에 디자인, 브라우저, 스크립트, 성능, 접근성 등과 긴밀한 관계가 있습니다.
  • 향후 업무 변경이나 버전업으로 인한 화면 추가/변경 시 다수의 퍼블리셔가 만든 코드의 일관성을 유지하고 서로간의 이해도를 높여 신속하게 대응할 수 있습니다.
  • 통일성 있는 코드로 인해 개발자와의 커뮤니케이션이 일관성 있게 진행 됩니다.
  • 불필요한 리소스를 방지하고 소프트웨어 유지 및 운영에 효율적인 코드를 제공할 수 있습니다.

코드 작성 규칙(공통)

구분 가이드라인 비고
적용대상 OS Native : IOS 10.0 이상, Android 6.0 이상
WEB : 모바일 브라우저
개발환경 기본적으로 Editor는 VS Code를 사용한다.
CSS파일은 .css파일과 .min.css 두 가지 버전으로 배포한다.
HTML 유효성 검사는 W3C Validation extension(JDK필요)을 설치하여 실시간 검증한다. 마크업 오류 방지
형상관리는 SVN으로 한다. (TortoiseSVN-1.14.0.28885-x64-svn-1.14.0)
공통 파일명은 Screen ID로 한다.
파일 명(html, css, image)은 Snake case를 사용한다. 언더스코어('_')를 사용 ui_common_pub.css
클래스 명은 Kebab case를 사용한다. 하이픈('-')을 사용 .custom-table
JavaScript에서 함수 및 변수명은 Camel case를 사용한다. popupOpen()
인코딩과 관련된 이슈를 방지하기 위해 '@charset'을 사용하여 UTF-8 인코딩을 강제한다. (다음에 빈 라인 1줄 추가)
파일의 마지막에는 빈 라인 1줄을 추가한다.
Indent는 에디터에서 Tab size 4, insert spaces 체크로 한다.
코드 정렬은 VS Code에서 Beautify extension을 추천한다. Prettier extension 보다 좀더 자유로움

HTML 작성 규칙

구분 가이드라인 비고
공통 HTML 마크업은 Semantic(의미론적)에 의거하여 작성합니다.
파일의 상단에 DTD를 표시한다. <!DOCTYPE html>
HTML 속성에는 기본적으로 큰 따옴표("")를 사용한다.
JavaScript에서는 작은 따옴표('')를 사용한다.
단일 태그(Single tag)는 뒤에 슬래시(/)를 생략한다. <br>, <input> 등
외부 css는 head 하단에, JavaScript는 body 하단에 배치한다.
버튼 onclick event가 필요한 상황이라면 버튼을 사용한다.
앵커 다른 리소스나 페이지 내의 위치를 가리키면 href 속성과 함께 앵커를 사용한다.
전화번호는 필히 <a href="tel:전화번호">전화번호<a> 링크를 걸어준다
주석 HTML 블록이 길어지는 경우 끝나는 지점 다음줄에 '// + 클래스 명'을 추가한다. <--// .wrapper -->
수정을 하는 경우 (날짜: 작업자 - 내용) 주석을 추가한다. <-- 2021-06-01 : 홍길동 - Textfield width 수정 -->
대체 텍스트 제공 이미지에는 반드시 alt 속성으로 적절한 대체 텍스트를 넣어준다.
대체 텍스트의 내용이 긴 경우는 대체 텍스트는 비워놓고 마크업으로 제공한다. (차트등 복잡한 콘텐츠 포함) .. alt=""><span class="sr-only">대체텍스트<span>
적절한 링크 텍스트 이미지를 독립적 링크로 제공하는 경우 alt에 적절한 링크 텍스트를 제공한다. <a href="#"><img src="btn.png" alt="다음버튼"></a>
링크 아이콘과 같이 링크에 의미 있는 배경 이미지를 적용한 경우에도 적절한 링크 텍스트를 제공한다. <a href="#" class="배경이미지">대체텍스트<a>
의미가 없는 이미지는 alt 속성에 빈 값을 제공한다.
OR코드의 경우 링크 주소를 알 수 있도록 대체 텍스트를 제공한다. alt="https://www.kiwoom.com 바로가기 OR 코드"
링크의 용도나 목적을 알 수 있도록 적절한 링크 텍스트를 제공한다. <span class="sr-only">공지사항<span>더 보기
레이블 제공 사용자 입력에는 이에 대응하는 레이블을 제공한다. Label의 for값과 id값을 동일하게 제공
label이 없는 input, select 태그는 title 속성을 넣어준다. <input type="text" title="생년월일 중 월 입력">

CSS 작성 규칙

구분 가이드라인 비고
CSS 클래스명은 기본적으로 'OOCSS(Object Oriented CSS)' 방식을 따릅니다. 일명 레고방식 or Bootstrap 방식
클래스 명은 의미를 파악하기 어렵게 함부로 축약하지 않는다.
ID 선택자는 특별한 경우를 제외하고 사용하지 않는다. Library 사용 시 개발자와 협의 필요
전체 선택자와 태그 선택자의 사용을 지양한다.
모든 속성은 영문 소문자로만 작성한다.
가급적 과도한 중첩은 사용하지 않는다. (최대 5Depth)
의미 없는 줄 바꿈은 사용하지 않는다.
선택자의 그룹핑이 필요할 때는 줄 바꿈 한다.
클래스 명 선언 후 여는 줄괄호 앞에 띄어쓰기 1칸을 추가하고 닫는 중괄호는 줄 바꿈 한다. .class-name {}
콜론(:) 후에는 띄어쓰기 1칸을 추가하고, 속성값 마지막에는 세미콜론(;)을 반드시 쓴다. width: calc(100% - 10px);
한 줄에 하나의 속성값만 선언한다. 속성값이 길어지는 경우 쉼표 뒤에 줄 바꿈을 사용할 수 있다.
소수 값에는 0을 표시한다. 후행 0은 표시하지 않는다.
0은 단위를 생략한다.
background의 url 속성의 파라미터 값에는 따옴표를 생략한다.
background-position은 숫자로 선언한다 background-position: 50% 50%;
컬러값은 축약형을 사용한다. color: #ffffff (X) > color: #fff (O)
border가 없는 경우 'none' 대신 0을 사용한다. outline은 'none'을 사용한다. border: none (X) > border: 0 (O)
media query 속성은 인접하여 선언한다. (그룹핑으로 관리가 필요한 경우 협의하여 찾기 쉬운 곳에 선언)
속성을 정의 할 때의 순서는 따로 지정하지 않지만 가급적 비슷한 속성들끼리 붙여서 작성한다. width와 height, padding과 margin 등
